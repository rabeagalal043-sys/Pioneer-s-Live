# Pioneer's Live Streaming

منصة عرض بث/محاضرات بتصميم استوديو بث، بتاخد لينك يوتيوب (فيديو عادي / بريمير / لايف) وتعرضه في شاشة كبيرة بكونترولز خارجية بسيطة (تشغيل/إيقاف — جودة — ملء الشاشة)، وتسجيل الدخول بيكون برقم الموبايل المسجل فقط.

## 📁 هيكل المشروع

```
pioneer-live/
├── index.html          صفحة تسجيل الدخول (رقم الموبايل)
├── player.html         شاشة عرض البث
├── css/style.css       التصميم
├── js/
│   ├── config.js       ← هنا بتحدّد لينك اليوتيوب الحالي
│   ├── login.js        منطق تسجيل الدخول
│   └── player.js       منطق البلاير والحماية
└── data/
    └── students.json   قاعدة بيانات الطلاب (الاسم + الموبايل)
```

## ▶️ التشغيل

المشروع Static بالكامل، لكن لازم يتفتح عن طريق **سيرفر محلي** (مش بفتح الملف مباشرة دبل كليك)، لأن المتصفح بيمنع قراءة `students.json` بـ `fetch` من على نظام الملفات مباشرة (CORS).

أسهل طريقة (لو عندك Python):

```bash
cd pioneer-live
python3 -m http.server 8080
```

وبعدين افتح: `http://localhost:8080`

أو لو عندك VS Code: استخدم إكستنشن **Live Server**.

عند رفع المشروع على أي استضافة عادية (Hostinger, Netlify, GitHub Pages, إلخ) هيشتغل عادي جدًا من غير أي تعديل.

## 👨‍🎓 إضافة/تعديل الطلاب

افتح `data/students.json` وزوّد العناصر بنفس الشكل:

```json
{ "name": "اسم الطالب", "phone": "01xxxxxxxxx" }
```

## 🎥 تغيير لينك المحاضرة

افتح `js/config.js` وغيّر `youtubeUrl` بس. بيقبل:

- `https://www.youtube.com/watch?v=VIDEO_ID` (فيديو عادي أو بريمير)
- `https://youtu.be/VIDEO_ID`
- `https://www.youtube.com/live/VIDEO_ID` (لايف)
- `https://www.youtube.com/embed/VIDEO_ID`

⚠️ لينكات على شكل `youtube.com/@channel/live` (بدون ID مباشر) لا تُستخرج تلقائيًا من المتصفح — هات لينك الفيديو/اللايف نفسه من داخل الصفحة (فيه ID من 11 حرف في العنوان).

## 🔒 عن الحماية (مهم تعرفه كـ مسؤول المنصة)

اتعمل أقصى تأمين ممكن على مستوى الواجهة:

- الفيديو بيتحمّل من `youtube-nocookie.com` وبدون أي واجهة تحكم من يوتيوب نفسه (`controls=0`, `modestbranding`, `rel=0`, `disablekb=1`).
- فوق الفيديو تمامًا طبقة شفافة (`pointer-events: none` على الآيفريم + شيلد فوقه) بتمنع أي كليك يوصل مباشرة لواجهة يوتيوب، وبتمنع الدبل-كليك والرايت-كليك.
- التشغيل/الإيقاف والجودة بيتم بالكامل عن طريق YouTube IFrame API من الكونسول الخارجي فقط — الطالب ملوش أي تفاعل مباشر مع الفيديو نفسه.
- مفيش أي لينك أو زرار في الصفحة بيوجّه لموقع يوتيوب.

**بصراحة تامة:** ده أقصى تأمين ممكن على مستوى الواجهة الأمامية (Frontend) لأن الفيديو في النهاية بيتحمّل عن طريق آيفريم رسمي من يوتيوب (وده الشرط الوحيد المسموح به لعرض فيديوهات يوتيوب أصلًا). مستخدم متقدم جدًا وعنده خبرة تقنية عالية (زي فتح أدوات المطور وتتبّع الشبكة) يقدر نظريًا يوصل لرابط الفيديو. الطبقة دي بتمنع أكتر من 99% من محاولات الوصول العادية للطلاب، لكنها مش بديل عن تشفير فيديو حقيقي (DRM) اللي محتاج خدمة استضافة فيديو مخصصة (Vimeo Pro, Bunny Stream, Mux...) لو حابب مستوى حماية أعلى من ده.

## ✏️ الاسم

الاسم الظاهر فوق شاشة البلاير هو اسم الطالب اللي دخل برقمه، مسحوب من `students.json`.
